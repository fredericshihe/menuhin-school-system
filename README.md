# Menuhin School System

练琴跟踪系统单页应用（`index.html`）的维护文档。项目目标是让老师/管理员在手机与桌面端都能实时查看学生练琴状态，并在列表页、详情页、排行榜等页面获得稳定、流畅的体验。

## 主要功能

- 实时总览：显示在练/未练人数比例与核心状态。
- 学生列表：按分类展示学生卡片，支持进入详情页。
- 学生详情：展示当前状态、今日时间轴、近七天练琴汇总。
- 实时同步：通过 Supabase Realtime + 兜底轮询保证状态更新。
- 排行榜与说明：展示周榜、规则说明与评分详情。

## 排行榜空态说明（2026-03-23）

- `index.html` 的“本周排行榜”现在采用“显示空态提示”而不是整块隐藏。
- 若 `get_weekly_leaderboards()` 在周初返回空数组，页面会明确提示“等待本周第一批工作日练琴记录”。
- 若 RPC 调用失败，页面会直接显示“加载失败”提示，方便区分“真没数据”和“后端异常”。

## 本次性能与流畅度优化（2026-03-22）

本次优化不删减现有功能，重点降低“列表/详情滚动卡死、实时更新卡顿、异常抖动”的风险。

### 1) 列表页滚动与卡片更新优化

- 列表卡片“实时分钟数”刷新改为优先处理视口内卡片（最多 48 张），避免离屏无效计算。
- 修复列表分钟数计算中的重复重算问题（同一学生同一轮只计算一次当日数据）。
- 学生卡片渲染改为 DOM 节点复用 + `DocumentFragment` 重排，减少 `querySelector` 热点开销与重排成本。
- 去掉卡片内联 `onclick` 字符串绑定，改为事件监听，减少动态字符串解析成本。

### 2) 详情页实时链路稳定性优化

- 修复详情页实时更新判断字段不一致问题：统一使用 `uiState.selectedStudent`。
- 避免 WebSocket 重连后心跳定时器累积：新增心跳定时器清理逻辑，降低后台定时器泄漏风险。
- 关闭默认调试日志开关，减少高频 `console` 输出对主线程和滚动帧率的影响。

### 3) 动画与样式性能优化

- 将 `transition: all` 改为精准属性过渡（`transform/opacity/background-color/box-shadow`），减少不必要样式计算。
- 为列表容器补充 `contain` 与 `backface-visibility`，降低复杂视图切换时的绘制压力。
- 新增 `prefers-reduced-motion` 适配，保障低性能设备与系统无障碍设置下的流畅度。

## 验证建议

### 手工体验验证

- 在“实时练琴卡片列表页”连续上下滑动 30-60 秒，观察是否仍出现卡死。
- 在“学生详情页”上下滑动并保持实时更新 2-3 分钟，观察时间轴与当前时长是否稳定刷新。
- 频繁切换 主页 -> 列表 -> 详情 -> 返回，检查动画是否平滑、无闪烁和跳顶。

### 实时链路验证

- 在后台触发学生状态变更（assign/clear/enter/exit），确认列表与详情都可在可接受延迟内更新。
- 网络短暂断开后恢复，确认页面可以继续实时更新，且没有明显性能衰减（避免心跳累积）。

## 维护原则

- 保留现有功能行为，不做与业务规则无关的逻辑删改。
- 优先做“增量更新、避免全量重绘、减少无效计算”。
- 新增功能或规则时，优先复用现有函数，避免重复实现。

## 第二轮深度风险排查（2026-03-22）

从“长期运行稳定性”角度新增排查并修复以下高风险点：

- **心跳监听堆积风险**：原心跳逻辑在每次检测中动态注册通道监听，存在长期运行后监听器增多风险。已改为“通道状态 + 轻量探测”模型，避免监听泄漏。
- **心跳频率调整失效**：心跳频率变化后旧定时器未重建。已在频率变更时自动重建心跳定时器。
- **后台资源消耗过高**：页面隐藏时仍可能保留多个高频任务。已在 `visibilitychange` 中统一暂停/恢复列表刷新、详情时长刷新、周图刷新、时间轴刷新和心跳监控。
- **详情页滚动跳顶风险**：实时刷新流程下可能触发详情页 `scrollTop` 重置。已改为仅在“真实视图切换”时重置滚动位置。
- **手势跟手渲染抖动**：触摸移动事件中直接高频写样式。已改为 `requestAnimationFrame` 节流写入，降低主线程压力。
- **排行榜刷新风暴风险**：clear 事件密集时可能短时间触发多次刷新。已增加防抖，合并为单次延迟刷新。
- **隐藏页兜底轮询浪费**：已在兜底轮询中跳过隐藏页面，降低后台无效请求。

## 第三轮滚动卡死与底部遮挡修复（2026-03-22）

针对“轻触后上滑卡死”和“移动端底部内容被遮挡”进行根因修复：

- **手势冲突修复（核心）**：
  - 重构 `setupSwipeGestures`，从“touchstart 即接管”改为“先判定横向意图再接管”。
  - 仅在明确横向滑动阈值后才执行页面平移，纵向滑动立即放行给原生滚动。
  - `touchmove` 在拖拽态使用 `passive:false + preventDefault`，避免滚动与自定义平移竞争导致卡顿。
  - 补全手势取消/结束时样式清理，避免残留 transform/will-change 影响后续交互。

- **底部遮挡修复（核心）**：
  - 引入 `safe-area` 变量：`--safe-top`、`--safe-bottom`。
  - 关键视图从 `100vh` 升级为 `100dvh`（并保留兼容），修复 iOS 动态地址栏导致的高度误差。
  - 为 `body`、`#homeView`、`.list-view`、`.detail-view`、`.student-grid`、`.detail-content` 增加底部安全区内边距，防止内容落入 Home Indicator 区域。
  - 头部区域增加顶部安全区偏移，避免刘海屏贴边观感与误触。

## 第四轮列表底部滚动假死修复（2026-03-22）

针对“学生列表页滑到底后轻触再上滑，页面卡死且右侧滚动条突然变长”的问题，补充根因修复：

- **滚动链泄漏修复（核心）**：
  - 为 `.list-view`、`.detail-view` 增加 `overscroll-behavior-y: contain`，阻断滚动链传递到根页面。
  - 增加 `touch-action: pan-y`，明确列表/详情主手势为纵向滚动。

- **根页面滚动锁定（核心）**：
  - 新增 `lockRootScroll()` / `unlockRootScroll()` / `syncRootScrollLock()`。
  - 当进入列表/详情/排行等浮层式视图时，锁住 `html/body` 根滚动，并保存原始滚动位置。
  - 返回首页时恢复根滚动，避免列表到底后继续把底层主页文档滚动起来。

- **问题现象解释**：
  - 之前列表页到底后继续上滑，滚动会传递到根页面，右侧显示的长滚动条其实是底层整页文档的滚动条，不是列表本身的滚动条。
  - 这会造成视觉上“页面卡死、滚动条长度异常、列表像失去响应”，本轮修复就是专门切断这条错误滚动链。

## 第五轮弹窗层级与下层滚动锁定（2026-03-22）

针对“弹窗打开时仍能滚动下层页面”的问题，完成统一治理：

- **统一锁滚机制（核心）**：
  - 新增 `acquireModalScrollLock()` / `releaseModalScrollLock()`，与现有视图锁统一到 `syncRootScrollLock()`。
  - 当前视图不在首页，或任意弹窗处于打开状态时，都会锁住 `html/body` 根滚动。
  - 只有“无弹窗 + 首页视图”时才恢复根滚动。

- **已接入锁滚的弹窗**：
  - 排行榜规则弹窗（`rankingInfoModal`）
  - 音符币流水弹窗（`coinHistoryModal`）
  - 评分详情弹窗（`lb-modal`）
        - 旧学生信息弹窗（`modalOverlay`，兼容路径）

- **附带修复**：
  - 管理员验证弹窗的 ESC 监听改为可回收，避免关闭后键盘监听残留。
  - 排行规则弹窗改为统一 `openRankingInfoModal()` / `closeRankingInfoModal()`，避免分散的内联开关造成状态不同步。
